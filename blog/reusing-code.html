REUSING CODE IN HTML

The Problem

the concept is simple, you have some code, for instance a segment for the top menu-bar of every page, inclduding things like your social links, your Home button and your blog. you paste this code into every page you make, after all, you gotta write html for each page individually, but then you want to change it, so you have to go throug every page on your site and implement the same change. this work is tedious, super boring and takes a long time once your project grows a bit. so what can we do about it?


A possible fix

one way to solve this is by re-using code. re-using code means that we have a single source for the code, a single file, and then whenever we need to use that code, we refer back to our single source, that way we can update the source and see it take effect all over our site. this sort of recycling is common in many forms of programing, and in all kinds of work oustide computers as well, from art to industrial mechanics, re-usability and work-reduction are essential for managing large tasks, and a neat way to keep even small projects easy and maintainable


How i did it in HTML and JS

The premise

first off we need a example to work with, for this explenation to make sense, so lets use this one here, say i have a website with a header, a footer, and a body. the code for that might be stored in a file called "index.html" and look something like this :

<div id="header">
<button>Socials</button>
<button>home</button>
<button>Blog</button>
</div>

<div id="content">
<h1>things i love about cheese</h1>
<p>i love cheese</p>
</div>

<div id="footer">
<a href="example.com">My twitter</a>
<p>plz follow my twitter</p>
</div>

what we have here are three DIV tags, each of them for a part of the site, a header with three buttons, a footer with a link and some text about my twitter, and a body with the content of the page. now what we want to do, is to seperate the header and footer into each their own files, and then we want to refference those other files when we need to get the code for the respective parts


Seperating the files

Let us make 2 new files, one for the header and one for the footer, all we have to do is copy the entire div for each of those parts into a seperate HTML file, so "header.html" looks like this :

<div id="header">
<button>Socials</button>
<button>home</button>
<button>Blog</button>
</div>

and then "footer.html" looks like this :

<div id="footer">
<a href="example.com">My twitter</a>
<p>plz follow my twitter</p>
</div>

Loading files with JS

now all we have to do is simple, our index-file looks like this after we removed the header and footer "index.html" :

<div id="content">
<h1>things i love about cheese</h1>
<p>i love cheese</p>
</div>

at the top and bottom of the file we will add new divs that are placeholders for the snippets we seperated into their own files by placing an empty div with an apropriate ID in the place where we want the code :

<div id="header"></div>

<div id="content">
<h1>things i love about cheese</h1>
<p>i love cheese</p>
</div>

<div id="footer"></div>

now then, to add in our code when the site loads we will use a bit of javascript, first we gotta load in jquerry, which we can get from the jquerry CDN. after that we can use the script-tag to add load-commands for each of our snipptes. the command looks like this :

$(function () {
     $("#id").load("file.html");
}

what we are doing here is definig a functino where we want to get an element with a specific ID, then replace that with html loaded from a new file. the complete snippet looks something like this :

<div id="header"></div>

<div id="content">
<h1>things i love about cheese</h1>
<p>i love cheese</p>
</div>

<div id="footer"></div>

<script>
     $(function () {
          $("#header").load("header.html");
          $("#footer").load("footer.html");
     });
</script>

Summary

Reusing code and making modular code is in general adventagous as it prevents us from having to repeat our writing. it provides a great way to update your project in a managable way and lets you work on elements of your site instead of single pages. modularity when implemented correctly is flexible, and is in general not that hard to do right either
